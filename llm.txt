# mcp-framework

> Rust framework for building MCP (Model Context Protocol) servers. Wraps `rmcp` to handle transport, auth, CLI, and tracing.

## Dependency

```toml
[dependencies]
mcp-framework = { git = "https://github.com/Plawn/mcp-framework" }
rmcp = { version = "0.5", features = ["server"] }
tokio = { version = "1", features = ["full"] }
anyhow = "1"
```

Requires Rust nightly.

## Core API

The entire public API:

```rust
// Re-exports from lib.rs
pub use auth::{AuthProvider, BasicAuthConfig, OAuthConfig, TokenStore};
pub use capability::{CapabilityFilter, CapabilityRegistry};
pub use runner::{run, LogLevel, McpApp, Settings, TransportMode};
```

## Minimal example

```rust
use mcp_framework::{run, McpApp, AuthProvider};
use rmcp::ServerHandler;

struct MyServer;

// Implement rmcp::ServerHandler for your server.
// See rmcp docs for #[tool], #[prompt], #[resource] macros.
impl ServerHandler for MyServer {
    fn name(&self) -> String { "my-server".to_string() }
}

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    run(McpApp {
        name: "my-server",
        auth: AuthProvider::None,
        server_factory: |_token_store| MyServer,
        stdio_token_env: None,
        settings: None,
        capability_registry: None,
        capability_filter: None,
    }).await
}
```

## McpApp fields

```rust
pub struct McpApp<F> {
    pub name: &'static str,           // App name, used in OAuth templates and logs
    pub auth: AuthProvider,            // Auth mode for HTTP transport
    pub server_factory: F,             // Fn(TokenStore) -> impl ServerHandler
    pub stdio_token_env: Option<&'static str>, // Env var name for stdio token (e.g. "MY_TOKEN")
    pub settings: Option<Settings>,    // None = CLI + env vars, Some = manual config
    pub capability_registry: Option<CapabilityRegistry>, // Dynamic tools/prompts/resources
    pub capability_filter: Option<Arc<dyn CapabilityFilter>>, // Per-session capability filtering
}
```

## Settings (programmatic config)

When `settings` is `Some(...)`, no CLI parsing happens and `.env` is not loaded.

```rust
use mcp_framework::{Settings, TransportMode, LogLevel};

Settings {
    transport: TransportMode::Http,     // Http, Sse, or Stdio
    log_level: LogLevel::Info,          // Error, Info, Debug, Trace
    bind_addr: "0.0.0.0:4000".into(),  // HTTP listen address
    public_url: None,                   // None derives from bind_addr
}
```

`Settings` implements `Default` (Http, Info, 0.0.0.0:4000, None).

When `settings` is `None`, the framework parses CLI args (`--transport`, `--debug`, `--trace`) and reads `BIND_ADDR`/`PUBLIC_URL` env vars.

## Transport modes

- **Http** — Streamable HTTP transport (default). MCP endpoint at `/mcp` and fallback `/`.
- **Sse** — Server-Sent Events transport (legacy MCP). SSE endpoint at `/sse`, messages at `/message`.
- **Stdio** — stdin/stdout for local Claude Desktop integration.

## Authentication

### No auth

```rust
auth: AuthProvider::None,
```

### HTTP Basic

```rust
use mcp_framework::BasicAuthConfig;

// From struct:
auth: AuthProvider::Basic(BasicAuthConfig {
    username: "admin".into(),
    password: "secret".into(),
}),

// From env (BASIC_AUTH_USERNAME, BASIC_AUTH_PASSWORD):
auth: AuthProvider::Basic(BasicAuthConfig::from_env().unwrap()),
```

### OAuth 2.0 (Keycloak OIDC)

```rust
use mcp_framework::OAuthConfig;

// From struct:
auth: AuthProvider::OAuth(OAuthConfig {
    client_id: "my-client".into(),
    client_secret: "secret".into(),
    issuer_url: "https://keycloak.example.com/realms/myrealm".into(),
    redirect_url: "http://localhost:4000/oauth/callback".into(),
    scopes: vec!["openid".into(), "profile".into(), "email".into()],
}),

// From env (OAUTH_CLIENT_ID, OAUTH_CLIENT_SECRET, OAUTH_ISSUER_URL, OAUTH_REDIRECT_URL, OAUTH_SCOPES):
auth: AuthProvider::OAuth(OAuthConfig::from_env().unwrap()),
```

OAuth tokens are automatically refreshed when expired (if a refresh token and config are available).

## TokenStore

The `server_factory` closure receives a `TokenStore`. Use it to access the authenticated user's token inside your server handler:

```rust
use mcp_framework::TokenStore;

struct MyServer {
    token_store: TokenStore,
}

// In McpApp:
server_factory: |token_store| MyServer { token_store },
```

Key methods:
- `token_store.get_token(session_id).await -> Option<StoredToken>` — get stored token (auto-refreshes if expired)
- `token_store.has_valid_token(session_id).await -> bool` — check if non-expired token exists
- `stored_token.access_token` — the access token string

## Capability system

Use `CapabilityRegistry` to dynamically add/remove tools, prompts, and resources at runtime:

```rust
use mcp_framework::CapabilityRegistry;

let registry = CapabilityRegistry::default();
registry.add_tool(tool_info, handler).await;
registry.remove_tool("name").await;
```

Use `CapabilityFilter` to control per-session visibility based on the user's token:

```rust
use mcp_framework::CapabilityFilter;

// Implement the trait or use a closure to filter capabilities per session
```

## HTTP endpoints

In HTTP mode, the MCP protocol endpoint is at `/mcp`.
In SSE mode, the SSE endpoint is at `/sse` and messages are posted to `/message`.

With OAuth auth, additional endpoints are registered:
- `/.well-known/oauth-protected-resource`
- `/.well-known/oauth-authorization-server`
- `/oauth/register`, `/oauth/authorize`, `/oauth/token`
- `/oauth/login`, `/oauth/callback`, `/oauth/status`
